# Proveria V3.0.0 Release Notes

Release date: May 25, 2026

## Summary

V3 is the OCR and exact-image evidence release. It extends the V2 desktop-first
content proof workflow with local scanned-PDF OCR proof generation and
byte-for-byte PNG/JPEG exact image proof, while preserving the same privacy
boundary: source text, OCR text, source images, and source pixels stay local.

## What Shipped

- Local OCR fallback for scanned or unselectable-text PDFs.
- OCR-derived text-content proof leaves with method/version metadata.
- Desktop preview for OCR coverage, confidence, and low-confidence warnings.
- Verifier OCR passage lookup with explicit OCR match/no-match language.
- Public result pages and result PDFs that identify OCR content matches.
- Exact PNG/JPEG image proof using byte-for-byte SHA-256 component leaves.
- Desktop preview and attestation detail coverage for exact image proof.
- Verifier image upload for exact image match and no-match lookup.
- Public receipt and result artifacts that label exact image proof clearly.
- V3-focused human QA checklist, known limitations, and sign-off record.

## Known Limitations

- OCR proof is deterministic OCR text matching, not semantic similarity.
- OCR match quality depends on local OCR extraction quality.
- Exact image proof is byte-for-byte matching, not visual similarity.
- Resized, recompressed, cropped, color-adjusted, metadata-stripped, or
  format-converted images can fail exact matching even when they look the same.
- Perceptual image similarity is not part of V3.0.0 and is tracked in
  `docs/future-product-backlog.md` pending threshold policy and careful result
  language.
- Revoking verifier access blocks new lookups, but previously issued public
  verification artifacts remain historical records.
