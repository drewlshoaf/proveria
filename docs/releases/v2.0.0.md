# Proveria V2.0.0 Release Notes

Release date: May 24, 2026

## Summary

V2 is the desktop-first content proof release. It adds local text-content proof
generation for plain text and native-text PDFs, verifier passage lookup, clearer
receipt/result artifacts, and a polished desktop workflow for producers,
admins, verifier access, records, and events.

## What Shipped

- Desktop-first producer workflow with improved projects, attestations,
  requests, account, and events surfaces.
- Browser-side whole-file SHA-256 hashing, external hash submission, and
  multi-file attestation submission.
- Local content proof coverage for plain text and native-text PDF files.
- Verifier lookup support for files, pasted SHA-256 values, and text passages.
- Public result pages and PDF/JSON artifacts for verifier lookup outcomes.
- Public attestation receipt pages and PDF/JSON receipt artifacts.
- Verifier access grants, revocation, fallback access requests, approval,
  denial reasons, and editable handoff messages.
- Compact attestation detail layout with Records, Verifications, and Events.
- Searchable, sortable, pageable, filterable, and resizable management tables.
- Human QA checklist and future product backlog documentation.

## Known Limitations

- V2 content proof is deterministic text matching, not semantic similarity.
- Native PDF matching depends on browser/pdf.js text extraction order.
- Short passages can fail to generate a content proof hash; use at least seven
  words from one continuous passage.
- Scanned PDFs, OCR-derived text, images, visual similarity, and perceptual
  hashes are outside V2 and planned for later work.
- Revoking verifier access blocks new lookups, but previously issued public
  verification artifacts remain historical records.
